會議主題

會議主要探討了 NVIDIA 在智能座艙領域的技術發展，包括全鏈路加速生成式 AI 應用的開發、大模型推理框架的優化，以及端側部署方案的實現。會議重點關注了如何利用 NVIDIA 的技術棧構建高效、穩定的生成式 AI 應用，並在車端實現大模型的快速推理。

主要技術點

  • NEMO 平台： 提供端到端的全流程解決方案，涵蓋數據處理、分布式訓練、模型客製化、推理加速、RAG 以及安全護欄等各個階段。
  • Curator 數據處理 Pipeline： 包含文本、圖片和視頻預處理，利用 Rapids 和 Dali 等 GPU 加速庫提高處理速度，並通過去重、過濾等步驟形成高質量數據集。
  • FP8 訓練： 通過 Transformer Engine 庫實現，利用 E4M3 和 E5M2 兩種格式，在保證精度的前提下，提高算力，縮短訓練時間，並節省顯存。
  • TRT-LM 框架： 針對大語言模型推理進行了深度優化，包括高效 Runtime、深度優化的 Kernel、多種大模型支持、自定義 Workflow 以及高級量化方法等。
  • 多模態大模型部署： 針對 VLM 和 DIT 模型，提供了靈活的分布式部署方案和性能優化方法，例如分離式部署、張量並行、序列並行以及 FP8 量化等。
  • DriveOS LLM SDK： 為車端量身定制的大模型推理框架，基於 C++ 開發，滿足車規功能安全要求，具有輕量化、易上手、功能豐富等優勢，並針對車端常用模型進行了專門優化。

決策與共識

  • 採用 NVIDIA NEMO 平台構建生成式 AI 應用。
  • 利用 Curator 數據處理 Pipeline 準備高質量數據集。
  • 通過 FP8 訓練加速大模型訓練過程。
  • 使用 TRT-LM 框架優化大模型推理性能。
  • 針對多模態大模型，採用靈活的分布式部署方案和性能優化方法。
  • 在車端部署 DriveOS LLM SDK，實現大模型的快速推理。

時間規劃與里程碑

  • 計劃支持更多的模型，例如千問 2.5VL、千問 2.5VLEnternVLDeepSeq 蒸餾的模型等。
  • 持續優化 INT4 的 prefuel 階段性能，並實現相關的 NVFP4 Kernel。

  • 擴展 DriveOS LLM SDK 的模型支持，增加對更多模型的支持。
