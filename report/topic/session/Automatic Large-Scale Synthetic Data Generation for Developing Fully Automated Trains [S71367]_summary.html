會議主題

會議主要探討了如何利用 NVIDIA Omniverse 和其他先進工具，構建一個自動化、可擴展的模擬管線，用於生成大規模的合成數據，以支持全自動列車的開發、測試和驗證。

主要技術點

  • 場景定義 (Scenic)： 使用 Scenic 語言定義邏輯場景，包括各種選項和範圍，從而生成具體的模擬場景。
  • 數據採樣： 從邏輯場景中採樣生成具體場景，定義列車和障礙物在 3D 空間中的精確位置、傳感器配置和列車模型。
  • 模擬執行 (iSEC SIM)： 使用 iSEC SIM 作為模擬引擎，在 NVIDIA OVX 上運行，模擬虛擬傳感器（相機、雷達、LiDAR）的數據採集過程。
  • 數據儲存 (MARV)： 將模擬生成的數據（ROSback 和 ASAM open label 格式）儲存在 MARV 數據管理平台中，用於後續的測試和機器學習。
  • 自動場景生成： 自動整合高程數據、氣象信息、鐵路基礎設施數據和 3D 模型，生成高質量的數字孿生環境。
  • ROS2： 使用 ROS2 作為機器人框架，實現不同組件之間的通信和數據交換。
  • NVIDIA Omniverse： 基於 NVIDIA Omniverse 構建模擬管線，利用其強大的渲染和物理模擬能力。

決策與共識

  • 採用基於 NVIDIA Omniverse 的模擬管線，以實現全自動列車開發所需的大規模合成數據生成。
  • 自動化場景生成流程，以提高效率和可擴展性。
  • 使用 iSEC SIM 作為模擬引擎，以確保模擬的準確性和真實性。
  • 將模擬數據用於測試和機器學習，以加速全自動列車的開發和驗證。
