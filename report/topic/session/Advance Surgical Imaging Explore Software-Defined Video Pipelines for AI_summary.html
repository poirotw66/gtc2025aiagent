會議主題

會議主要探討了如何利用軟體定義的影像處理管線，結合 AI 技術，提升手術影像的品質和效率，最終改善患者的治療效果。

主要技術點

  • 傳統硬體影像處理管線 (FPGA-based):
    • 優點：速度快、效能高，適合即時 4K 影像處理。
    • 缺點：靈活性低、擴展性差，難以適應新的相機晶片、AI 演算法和更高解析度。
  • 混合式影像處理管線 (FPGA + GPU):
    • 優點：部分處理在 FPGA 上完成，部分處理在 GPU 上完成。
    • 缺點：硬體依賴性高、架構複雜，難以實現即時資料傳輸。
  • 軟體定義影像處理管線 (Software ISP):
    • 優點：靈活性高、可擴展性強，易於整合 AI，開發週期短。
    • 缺點：需要高效能運算平台，才能實現即時處理。
  • 影像處理流程:
    • 從相機晶片獲取原始資料 (raw readings)。
    • 進行重新定時 (retiming)、黑階校正 (black level correction)、非均勻性校正 (non-uniformity correction) 等預處理。
    • 應用專有的濾波器 (secret sauce filters) 和去馬賽克 (demosaicing) 演算法。
    • 進行白平衡 (white balancing)、色彩抑制 (chromo suppression) 和色彩校正 (color correction)。
    • 進行曝光控制 (exposure control)，以適應手術過程中光線的變化。
  • NVIDIA IGX 和 Holoscan:
    • IGX 提供高效能運算平台，支援 AI 加速功能。
    • Holoscan SDK 提供易於使用的開發工具，方便開發客製化的影像處理運算元 (operators)。
    • ConnectX-7 晶片提供高速資料傳輸，支援將資料直接推送到 GPU 記憶體。
  • 客製化運算元 (Custom Operators):
    • 用於解包 (unpack) 原始資料、應用專有的濾波器和執行其他客製化的影像處理任務。
    • 可以利用 CUDA 核心 (CUDA kernels) 和 NVIDIA Performance Primitives (NPP) 來優化效能。
  • 延遲 (Latency):
    • 目標是實現低於 100 毫秒的光到光 (light to light) 延遲。
    • 目前的 Kallstolz 系統延遲約為 60 毫秒。
    • 使用 Holoscan 建立的簡單管線延遲約為 30 毫秒。

決策與共識

  • 軟體定義的影像處理管線是未來手術影像發展的趨勢。
  • NVIDIA IGX 和 Holoscan 平台是實現軟體定義影像處理管線的理想選擇。
  • 需要開發客製化的運算元，以滿足手術影像的特定需求。
  • 需要持續優化軟體和硬體，以降低延遲並提高效能。

時間規劃與里程碑
